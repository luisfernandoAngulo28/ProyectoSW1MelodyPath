#### 1.6.2 SOFTWARE

##### 1.6.2.1 Servidores

La arquitectura del lado del servidor requiere un conjunto robusto de tecnologías para soportar la carga de procesamiento y almacenamiento:

- **Lenguajes y Frameworks:** Se empleará Python apoyado en el framework Django, o de manera alternativa Node.js con Express, dada su idoneidad para procesar peticiones complejas y manejar la integración con modelos de Machine Learning.
- **Base de Datos y ORM:** Se utilizará PostgreSQL como sistema gestor de base de datos relacional, estructurando y consultando la información de usuarios y estadísticas a través de un ORM como Prisma o SQLAlchemy.
- **Inteligencia Artificial:** Se emplearán librerías especializadas como TensorFlow, PyTorch y Librosa para llevar a cabo el análisis de frecuencias, reconocimiento de acordes y detección de la altura del sonido (pitch) en tiempo real.
- **Seguridad:** El control de sesiones, protección de rutas y autenticación se gestionará de manera segura utilizando JSON Web Tokens (JWT).
- **Infraestructura:** Se hará uso de Docker para la configuración, encapsulamiento y despliegue temporal en entornos de pruebas y servicios en la nube.

##### 1.6.2.2 Clientes

El software ejecutado en el dispositivo del usuario está orientado a la web y no demanda instalaciones complejas:

- **Navegador Web:** La plataforma será completamente funcional y accesible desde navegadores modernos actualizados como Google Chrome, Mozilla Firefox, Microsoft Edge o Safari.
- **Frontend y Diseño UI:** La interfaz gamificada, interactiva y visualmente atractiva estará desarrollada con la librería React y el framework Next.js, apoyándose en bibliotecas de componentes de diseño como Shadcn o Material-UI.
- **Procesamiento Nativo:** Se utilizará de forma crítica la Web Audio API, una interfaz nativa del navegador para capturar, enrutar y procesar el audio en tiempo real directamente desde el cliente antes o durante la comunicación con el servidor.

##### 1.6.2.3 Otro Software adicional

El ciclo de vida del desarrollo y las operaciones del sistema se apoyarán en las siguientes herramientas complementarias:

- **Control de Versiones:** Todo el código fuente, trabajo colaborativo mediante ramas y control de revisiones se alojará en repositorios de GitHub.
- **Gestión de Proyectos:** La planificación, seguimiento de Sprints, historias de usuario y el Product Backlog se gestionarán mediante los tableros visuales de Jira bajo el marco de trabajo Scrum.
- **Modelado y Diseño:** Se utilizará Enterprise Architect para la diagramación del Modelo C4, casos de uso y diagramas UML.
- **Entorno de Desarrollo (IDE):** El equipo de programación utilizará Visual Studio Code.
- **Integración de Medios:** El sistema utilizará de forma externa la API de YouTube o la API de Vimeo para integrar y reproducir dinámicamente los tutoriales en video dentro de la plataforma sin tener que alojarlos internamente.

#### 1.6.3 DATOS

El sistema estructura y gestiona múltiples entidades de información críticas para la experiencia educativa y la gamificación:

- **Usuario / Perfil:** Almacena la información de acceso e identidad, incluyendo identificador único, nombre, correo electrónico, credenciales seguras (contraseñas), el instrumento principal seleccionado por el estudiante y el nivel de habilidad inicial.
- **Progreso y Gamificación:** Registra el avance continuo del estudiante. Contiene métricas como los puntos de experiencia acumulados (XP), el nivel actual de progreso, insignias o logros desbloqueados, lecciones completadas, y el registro de rachas diarias.
- **Catálogo de Lecciones y Tutoriales:** Base de datos del contenido educativo organizado por disciplina (piano, guitarra, batería, canto) y segmentado por nivel de dificultad. Incluye contenido teórico, práctico y las referencias a medios audiovisuales (enlaces de video).
- **Resultados de Evaluación:** Almacena los reportes detallados del desempeño del usuario tras realizar prácticas o pruebas, incluyendo la puntuación obtenida, estadísticas de rendimiento y errores de precisión, ritmo o afinación detectados.

#### 1.6.4 PROCESOS

El comportamiento de MelodyPath se divide en procesos clave que garantizan la funcionalidad y la interacción con el usuario:

- **Proceso de Autenticación y Gestión de Perfil:** Permite a los usuarios realizar un registro seguro, iniciar o cerrar sesión, recuperar contraseñas olvidadas, y configurar parámetros iniciales como su instrumento de interés y nivel.
- **Proceso de Evaluación Diagnóstica:** Se ejecuta al ingresar al sistema por primera vez, analizando los conocimientos musicales del usuario para clasificar automáticamente su nivel inicial y personalizar su ruta de estudio.
- **Proceso de Aprendizaje Adaptativo:** Selecciona y ajusta dinámicamente el contenido de los tutoriales y lecciones basándose en el nivel del estudiante y los resultados previos de sus evaluaciones, desbloqueando contenido de forma progresiva.
- **Proceso de Análisis y Retroalimentación en Tiempo Real:** El sistema captura las señales de audio generadas por el usuario, las procesa mediante inteligencia artificial comparándolas con un patrón esperado, y emite correcciones visuales o auditivas inmediatas sobre afinación, acordes y ritmo.
- **Proceso de Gamificación y Comunidad:** Evalúa las acciones del usuario para otorgar puntos de experiencia (XP), actualizar los rankings semanales, asignar logros y registrar estadísticas, además de facilitar la redirección a grupos de interacción social.
- **Proceso de Administración:** Permite a los administradores gestionar los usuarios del sistema, crear, actualizar y organizar el contenido de las lecciones, monitorizar estadísticas de uso y gestionar los retos y recompensas.

## 2.1 Marco referencial

En los últimos años, las tecnologías digitales han transformado significativamente la forma en que las personas acceden al conocimiento y desarrollan nuevas habilidades. En el ámbito educativo, las plataformas de aprendizaje en línea han permitido que millones de usuarios puedan adquirir conocimientos de manera flexible, interactiva y accesible desde cualquier lugar del mundo.

Dentro de este contexto, el aprendizaje musical también ha experimentado una evolución importante gracias al uso de herramientas tecnológicas que facilitan la enseñanza de instrumentos y técnicas vocales mediante aplicaciones web y móviles. Estas plataformas combinan contenidos educativos, ejercicios interactivos y recursos multimedia que permiten a los estudiantes aprender a su propio ritmo.

Uno de los enfoques más utilizados actualmente en las plataformas educativas es la gamificación, una estrategia que consiste en aplicar elementos propios de los videojuegos dentro de entornos de aprendizaje. Elementos como puntos, niveles, recompensas, logros y desafíos ayudan a aumentar la motivación del usuario, promoviendo una mayor participación y compromiso con el proceso educativo.

El éxito de plataformas educativas basadas en gamificación ha demostrado que este enfoque puede mejorar significativamente la experiencia de aprendizaje. Aplicaciones como Duolingo, por ejemplo, han popularizado este modelo al convertir el aprendizaje de idiomas en una experiencia dinámica y entretenida, basada en retos progresivos y recompensas por el avance del usuario.

En el ámbito musical, la integración de tecnologías digitales permite desarrollar sistemas capaces de guiar al estudiante durante su proceso de aprendizaje mediante lecciones estructuradas, ejercicios prácticos y retroalimentación automática. Estas plataformas pueden incluir contenidos relacionados con teoría musical, reconocimiento de notas, práctica de acordes, ritmo y técnicas de interpretación de instrumentos.

Asimismo, el desarrollo de herramientas de análisis de audio ha abierto nuevas posibilidades para el aprendizaje musical asistido por computadora. Mediante algoritmos especializados, los sistemas pueden analizar el sonido producido por el usuario y determinar si las notas o ejercicios de canto se ejecutan correctamente, ofreciendo retroalimentación inmediata que permite mejorar el desempeño del estudiante.

Otro aspecto relevante es el uso de sistemas inteligentes de aprendizaje, los cuales permiten adaptar el contenido educativo según el progreso y desempeño del usuario. Estos sistemas pueden recomendar nuevas lecciones, ajustar el nivel de dificultad de los ejercicios y generar reportes de avance personalizados, contribuyendo a una experiencia de aprendizaje más eficiente.

En este contexto, el presente proyecto propone el desarrollo de una plataforma interactiva para el aprendizaje de instrumentos musicales y clases de canto, basada en técnicas de gamificación y apoyada por herramientas tecnológicas modernas. El sistema permitirá a los usuarios acceder a lecciones progresivas, realizar ejercicios prácticos y visualizar su progreso dentro de la plataforma.

De esta manera, la solución busca facilitar el acceso a la educación musical, ofreciendo una alternativa flexible y accesible frente a los métodos tradicionales de enseñanza. Además, el uso de tecnologías web modernas permitirá que la plataforma sea utilizada desde distintos dispositivos, ampliando las oportunidades de aprendizaje para personas interesadas en desarrollar habilidades musicales.

## 2.2 Marco de trabajo Ágil Scrum

### Propósito de Scrum

El propósito de Scrum es facilitar la colaboración y la organización de los equipos de desarrollo para entregar productos complejos de manera eficiente y adaptativa, enfocándose en generar valor continuo para los clientes. Scrum permite reducir la complejidad del desarrollo de software y responder de forma ágil a los cambios en los requisitos y necesidades de los usuarios.

### Visión general de Scrum

La metodología Scrum aplicada en el proyecto se estructura bajo los siguientes principios:

- El desarrollo se realiza mediante iteraciones llamadas sprints, en las que el equipo realiza entregas incrementales del producto.
- Se llevan a cabo actividades regulares como reuniones diarias (daily scrum) y reuniones de planificación y revisión de sprint, que garantizan la transparencia y la mejora continua.
- Cada entrega busca generar un incremento de producto completamente funcional, que pueda ser revisado por el cliente para obtener retroalimentación temprana.

### El Equipo Scrum (Scrum Team)

Scrum propone un equipo auto-organizado y multifuncional, capaz de gestionar sus tareas y comprometerse con los objetivos de cada sprint. El equipo se compone de los siguientes roles:

- **Product Owner:** Responsable de maximizar el valor del producto y gestionar el Product Backlog. Es el enlace entre el equipo de desarrollo y los interesados, garantizando que las prioridades del proyecto estén alineadas con las necesidades del cliente.
- **Scrum Master:** Encargado de asegurar que el equipo entienda y adopte los principios de Scrum. Es un facilitador que elimina obstáculos, promueve buenas prácticas y ayuda al equipo a mejorar su desempeño.
- **Equipo de Desarrollo:** Grupo de profesionales con distintas competencias (desarrolladores, diseñadores, analistas) responsables de transformar los elementos del Product Backlog en incrementos funcionales del producto.

### Eventos Scrum

Scrum define una serie de eventos para organizar el trabajo de manera efectiva:

- **Sprint:** Iteración de duración fija (generalmente de 2 a 4 semanas) durante la cual se desarrolla un incremento de producto potencialmente entregable.
- **Sprint Planning:** Reunión donde se define el objetivo del sprint y se seleccionan los elementos del Product Backlog a desarrollar.
- **Daily Scrum:** Reunión diaria de 15 minutos para sincronizar actividades y detectar impedimentos.
- **Sprint Review:** Revisión del incremento desarrollado para obtener retroalimentación del cliente y otras partes interesadas.
- **Sprint Retrospective:** Espacio para que el equipo reflexione sobre su desempeño y proponga mejoras para el siguiente sprint.

### Artefactos Scrum

- **Product Backlog:** Lista priorizada de tareas y funcionalidades requeridas para el producto.
- **Sprint Backlog:** Conjunto de elementos seleccionados del Product Backlog para un sprint, junto con el plan para entregarlos.
- **Incremento:** Conjunto de funcionalidades completadas durante el sprint, que cumplen con la definición de terminado y están listas para ser entregadas o revisadas.

### Ventajas y desventajas

**Ventajas:**
- Alta comunicación y coordinación entre los miembros del equipo.
- Entregas incrementales que permiten obtener resultados de valor rápidamente.
- Flexibilidad para adaptarse a cambios en los requisitos.
- Motivación del equipo gracias a la autonomía y el trabajo colaborativo.

**Desventajas:**
- Requiere un alto nivel de compromiso y comunicación continua por parte de todos los integrantes.
- Las reuniones y actividades pueden ser percibidas como demandantes en tiempo y esfuerzo.
- El avance del proyecto depende del trabajo conjunto y sincronizado de todo el equipo.

### Valores del trabajo Scrum

- Humildad
- Honestidad
- Respeto
- Compromiso
- Trabajo colaborativo

## 3 Herramientas tecnológicas para el desarrollo

### 3.1 Lenguajes de programación

**JavaScript / TypeScript (Frontend con Next.js)**

JavaScript es uno de los lenguajes de programación más utilizados en el desarrollo de aplicaciones web modernas debido a su flexibilidad, compatibilidad con todos los navegadores y amplio ecosistema de herramientas. En la actualidad, su evolución ha permitido la creación de aplicaciones web altamente interactivas, dinámicas y escalables.

Para el desarrollo de la interfaz del sistema se utilizará Next.js, un framework basado en React que permite construir aplicaciones web modernas con alto rendimiento, optimización automática y una arquitectura flexible. Next.js facilita la creación de interfaces dinámicas y rápidas, gracias a funcionalidades como renderizado del lado del servidor (SSR), generación de sitios estáticos (SSG) y renderizado híbrido, lo cual mejora la experiencia del usuario y el posicionamiento en buscadores.

El uso de Next.js permite organizar el proyecto mediante una estructura basada en componentes reutilizables, páginas y rutas automáticas, lo que facilita el mantenimiento y la escalabilidad del sistema. Además, se integra naturalmente con TypeScript, lo que permite agregar tipado estático al código, reduciendo errores y mejorando la calidad del desarrollo.

### 3.2 Frameworks y entornos de ejecución

**Frontend web**
- Next.js 14 para SPA y UI declarativa.
- Vite como bundler/dev server (rápido, DX moderna).
- Node.js 20 LTS + npm/pnpm para gestión de dependencias y scripts.

**Backend**
- Django 5.2 para capa web y administración.
- Django REST Framework para exponer APIs REST al frontend y a la app móvil.
- Entorno Python: venv + pip para aislar dependencias.
- Procesamiento/IA: OpenCV, Pillow, ffmpeg (imágenes/video); librosa (audio); transformers, scikit-learn (texto/código).
- Tareas asíncronas: Celery + Redis como broker.

**QA y soporte:** Postman/Insomnia para pruebas de API. Git/GitHub para control de versiones e integración continua (CI).

### 3.3 Sistema de gestor de base de datos

**PostgreSQL:** Es un sistema de gestión de bases de datos relacionales y de código abierto, reconocido por su robustez, flexibilidad y cumplimiento de estándares. Soporta consultas complejas y operaciones de gran volumen de datos, lo que lo convierte en una opción ideal para aplicaciones que requieren alta disponibilidad y escalabilidad. Su arquitectura permite compatibilidad con Python, Java, PHP, entre otros, y puede manejar JSON, XML y otros formatos avanzados.

### 3.4 Lenguaje de modelado de software (MODELO C4)

El modelo C4 es una técnica de notación gráfica para modelar la arquitectura de sistemas de software. Fue concebido por Simon Brown en 2006 y se basa en la descomposición estructural de un sistema en contenedores y componentes. Consta de cuatro niveles:

- **Diagrama de contexto del sistema:** Muestra cómo se ajusta el sistema al mundo que lo rodea, identificando los usuarios y sistemas externos con los que interactúa.
- **Diagrama de contenedores:** Muestra los bloques técnicos de alto nivel (aplicación web, base de datos, servidor de correo, etc.).
- **Diagrama de componentes:** Desglosa un contenedor individual en sus componentes internos (ej. carrito de compras, autenticación).
- **Diagrama de código:** Muestra cómo se implementa un componente en términos de clases, funciones y variables.

### 3.5 Herramienta de diseño y modelado

Enterprise Architect es una herramienta de software que ayuda a los equipos de desarrollo a visualizar, analizar y diseñar soluciones de software complejas. Ofrece diagramas UML, diagramas de flujo de datos y diagramas de entidad-relación. También permite la colaboración en tiempo real entre miembros del equipo y puede generar código a partir de los modelos creados.

- **El servidor de producción:** El entorno final donde se implementa y ejecuta la aplicación. Es importante mantener ambos entornos (montaje y producción) lo más similares posible.
- **El entorno de desarrollo integrado (IDE):** Herramienta que permite a los programadores escribir, generar, probar y depurar un programa en un mismo lugar. Se utilizará **Visual Studio Code**.

#### 4.1.7 Modelo inicial

##### 4.1.7.1 Modelo de contexto

El modelo de contexto representa una vista de alto nivel del sistema, tratándolo como una caja negra e identificando los principales actores externos que interactúan con él.

**Actores principales:**
- **Usuarios Estudiantes (Principiantes/Intermedios):** Personas que utilizan la plataforma para aprender instrumentos musicales o canto. Interactúan con lecciones, realizan ejercicios prácticos y reciben retroalimentación del sistema.
- **Administradores del sistema:** Encargados de gestionar el contenido educativo, actualizar lecciones, configurar niveles y supervisar el correcto funcionamiento de la plataforma.

**Entradas al sistema:**
- Datos de registro e inicio de sesión (credenciales del usuario).
- Selección de instrumento musical (piano, guitarra, batería o canto).
- Interacciones del usuario dentro de la plataforma (clics, navegación, avance en lecciones).
- Respuestas a ejercicios interactivos (selección de notas, acordes, ritmo, etc.).
- (Opcional) Señales de audio capturadas mediante el micrófono del usuario para análisis.

**Salidas del sistema:**
- Retroalimentación inmediata sobre ejercicios (correcto/incorrecto).
- Contenido personalizado según el nivel del usuario.
- Resultados de evaluaciones y progreso académico.
- Sistema de recompensas (puntos, niveles, logros).
- Estadísticas de desempeño y avance del usuario.
- Enlaces de acceso a comunidades externas (grupos de apoyo).

**Descripción del flujo general:**

El usuario se registra o inicia sesión en la plataforma y selecciona el instrumento que desea aprender. El sistema le asigna contenido acorde a su nivel (principiante o intermedio). El usuario accede a lecciones estructuradas y realiza ejercicios interactivos; el sistema analiza sus respuestas y proporciona retroalimentación inmediata. A medida que avanza, acumula puntos, desbloquea niveles y puede rendir evaluaciones. En caso de implementar el módulo de IA, el sistema podrá analizar audio en tiempo real. Finalmente, el usuario puede visualizar su progreso y acceder a comunidades externas para fortalecer su aprendizaje colaborativo.

#### 4.1.8 Definición de criterios de calidad

- **Fiabilidad:** Funcionamiento estable y continuo en todos los módulos, con disponibilidad mínima del 99%.
- **Precisión y validez de resultados:** Los algoritmos de IA deben alcanzar al menos un 85% de exactitud en el reconocimiento de notas, acordes y patrones rítmicos.
- **Usabilidad y latencia:** Interfaz intuitiva con latencia inferior a 2 segundos en ejercicios interactivos y tiempo de carga de lecciones no mayor a 3 segundos.
- **Seguridad y privacidad:** Comunicación mediante HTTPS, autenticación con JWT, y datos de audio no almacenados permanentemente sin consentimiento explícito del usuario.
- **Escalabilidad:** Arquitectura que permita el crecimiento horizontal en entornos cloud, soportando múltiples usuarios concurrentes sin afectar el rendimiento.
- **Mantenibilidad:** Arquitectura modular con separación clara entre interfaz, lógica de negocio y componentes de IA. Documentación técnica actualizada y cobertura de pruebas superior al 80% en módulos críticos.
